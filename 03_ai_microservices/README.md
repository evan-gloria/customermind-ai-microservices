# 🤖 03: AI Microservices Layer

This directory contains the core application logic of the Customer Intelligence Platform. 

To ensure high availability, fault tolerance, and secure isolation of API keys, the backend is built as a **Distributed Microservices Architecture** using FastAPI. Each service is highly decoupled, handling a specific domain of the AI reasoning or data retrieval process.

---

## 🏗️ Service Architecture

### 1. `orchestrator_service` (The API Gateway)
* **Role:** The central nervous system of the platform. The Streamlit frontend only ever communicates with the Orchestrator. 
* **Function:** It routes requests to the downstream microservices, handles cross-service state management, and aggregates the final JSON payloads to return to the UI.

### 2. `strategist_service` (The Reasoning Engine)
* **Role:** The primary generative AI engine, powered by **Vertex AI (Gemini 2.5)**.
* **Function:** Acts as a Tool-Augmented Microservice. It receives natural language prompts, utilizes Function Calling to securely trigger predefined tools (e.g., `query_database`, `fetch_ozbargain_deals`), parses live RSS feeds, and writes hyper-personalized marketing copy based on retrieved demographic data.

### 3. `reviewer_service` (The Compliance Auditor)
* **Role:** The safety and alignment layer, powered by **Llama 3.3 (via Groq)**.
* **Function:** Executes sub-second inference to audit the output of the Strategist service. It ensures the generated marketing campaigns do not violate brand safety guidelines, hallucinate non-existent products, or use exclusionary language. 

### 4. `data_modelling_service` (The BQ Sandbox)
* **Role:** The secure execution environment for the Text-to-SQL copilot.
* **Function:** It intercepts raw SQL generated by the Strategist, scrubs it for destructive keywords (`DROP`, `DELETE`), enforces a `LIMIT`, and securely executes the read-only query against the BigQuery Semantic Layer (`v_agent_semantic_layer`). 

### 5. `profiler_service` (The Math Engine)
* **Role:** The deterministic analytics layer.
* **Function:** Bypasses the LLM entirely to execute hard mathematical aggregations, cohort sizing, and affinity scoring based on the BigQuery ML K-Means clusters.

## 🧠 Key Engineering Decisions

### 1. Asynchronous UI Streaming
Standard LLM pipelines suffer from high latency, leaving users staring at loading spinners. This system utilizes FastAPI's `StreamingResponse` and `httpx` async generators to stream the step-by-step execution of the AI microservices directly to the Streamlit UI in real-time, drastically improving the perceived performance and UX.

### 2. Zero-Compute MLOps via Materialized Views
Running `ML.PREDICT` live on millions of rows for every UI click is expensive and slow. I implemented a background tool (`/tools/refresh-segments`) that:
1. Retrains the K-Means model on a 10% data sample.
2. Extracts centroid statistics and passes them to the Profiler LLM to generate dynamic semantic labels (e.g., "High-Value Tech Professional").
3. Injects those LLM labels back into a BigQuery `CREATE OR REPLACE TABLE` script, materializing a fully labeled, billion-row static table. 
*Result:* Frontend queries drop from seconds to milliseconds, and compute costs drop to near-zero.

### 3. Distributed Security
Internal microservices are blocked from the public internet and communicate strictly via zero-trust `X-API-Key` header validation, managed securely via Google Cloud Secret Manager.

## 🚀 Local Development

This project utilizes **Poetry** for strict dependency management and virtual environment isolation.

### Prerequisites
1. Ensure your master `.env` file at the root of the repository is configured.
2. Authenticate your local environment with Google Cloud (Application Default Credentials):
   ```bash
   gcloud auth application-default login
   ```
3. Install dependencies via Poetry:
   ```bash
   poetry install
   ```

### Running a Service Locally
To run a specific microservice for local testing or debugging, use Poetry to invoke Uvicorn. 

*Example: Running the Profiler Service*
```bash
poetry run uvicorn 03_ai_microservices.app.services.profiler_service.main:app --host 127.0.0.1 --port 8081 --reload
```
*(Adjust the module path and port number based on the specific service you are testing).*

## ☁️ Deployment (CI/CD)

This platform is designed to be cloud-native. Rather than relying on local Docker Compose networks, the microservices are deployed directly to **Google Cloud Run** using Google Cloud Deploy.

This ensures that each microservice scales independently (to zero when not in use) and secrets are injected natively via GCP Secret Manager at runtime.

---

## 🔒 Internal Security
All internal service-to-service communication is secured via a pre-shared master key. Downstream services (`strategist`, `reviewer`, `data_modelling`, `profiler`) will reject any internal HTTP requests that do not include the `X-API-Key` header matching the secret injected at runtime.
